# 自己搭桌宠运行时（Electron 极简挂件 + Agent hooks）

> 当你已经有一组做好的状态文件（SVG 路线 = `.svg.html`，APNG 路线 = `.apng`）和
> 一份 theme 映射（见 `state-map.md`），这条工作流教你把它们接成一个**真正在桌面上
> 跑、会随 AI agent 活动切状态**的极简运行时。
>
> 跨路线通用——运行时只消费"状态名 → 文件"的映射，不关心文件是矢量还是帧动画。
>
> 这条不替代现成运行时（clawd-on-desk / 各类 pet host）。它的价值是：当你想要
> **100% 自己的、零黑盒、可随手改**的运行时时，给你一套已经踩平坑的最小骨架。

---

## 适用判断

先确认你真的需要自己搭，而不是套现成的：

- **套现成运行时** —— 你只想尽快看到桌宠动起来，不在乎运行时是别人的；
- **自己搭（这条）** —— 你想完全掌控窗口行为 / 事件映射 / 部署，或想把"搭运行时"
  本身当作一次可复用的工程经验沉淀。

最小骨架只要 4 块，下面逐块说。

---

## 架构：最小可行运行时的 4 块

```
┌─────────────────────────────────────────────┐
│  主进程 (main)                               │
│   · 透明 / 无边框 / 置顶窗                    │
│   · 原生拖动 + 拖角缩放 + 尺寸记忆            │
│   · 本地 HTTP 控制服务（听 127.0.0.1:<port>）│
│   · 空闲/睡眠计时、通知后等用户回来           │
└───────────────┬─────────────────────────────┘
                │ IPC: send('pet-state', name)
┌───────────────▼─────────────────────────────┐
│  渲染层 (一个 HTML)                          │
│   · 用 iframe 加载当前状态文件               │
│   · 注入透明背景 + 尺寸变量                   │
│   · 瞬时态（happy/error）闪几秒自动回 idle    │
└─────────────────────────────────────────────┘
        ▲
        │ HTTP: GET /<state>
┌───────┴─────────────────────────────────────┐
│  Agent hooks（在你的 AI agent 配置里）       │
│   事件 → curl 127.0.0.1:<port>/<state>       │
└─────────────────────────────────────────────┘
```

四块的职责边界要清楚：**窗口行为在主进程，角色表演在状态文件，事件翻译在 hooks。**
别让任何一块越界（典型越界：为了模拟窗口推出，让角色在 SVG 内整体乱跳——那是窗口的活）。

---

## 工作流

### 第 0 步：前置

你需要先有：

- 一组状态文件（至少 `state-map.md` 的"必做 5 个"：idle / typing / thinking / sleeping / happy）；
- 一份 theme 映射（状态名 → 文件路径）；
- 状态文件自包含、能被直接加载（脚本状态不要靠普通 `<img>` 嵌入，见 `state-map.md` §Scripted SVG 嵌入）。

### 第 1 步：窗口外壳（主进程）

一个透明置顶无边框窗，把整窗交给角色：

- `transparent: true, frame: false, alwaysOnTop: true, hasShadow: false`；
- **移动**用系统原生拖动（CSS `-webkit-app-region: drag`），不要自己监听 mousemove 算偏移——会卡、会漂；
- **缩放**用角上的手柄（`-webkit-app-region: no-drag` + 指针捕获），把对角固定、跟手缩放；
- **尺寸记忆**：把当前尺寸写进运行时用户数据目录的 `config.json`，下次启动读回。

> 渲染层要拿到窗口尺寸去渲染像素艺术时，主进程先 `sendSync` 给个初值，渲染层兜底
> 一个默认值，**绝不让尺寸为 undefined**（像素艺术按尺寸算 viewBox，undefined 会整只崩）。

### 第 2 步：状态加载层（渲染层）

一个 HTML，收到状态名就换角色：

- 用 `iframe.srcdoc` 加载状态文件，并在 `</head>` 前注入 `html,body{background:transparent}`
  和尺寸变量（`--pixel-size` 之类）——这样状态文件本身不用关心透明和尺寸；
- 尺寸变化时：先快路径改 CSS 变量，再 `setTimeout` 兜底重渲染一次（防打包后被裁）；
- iframe 设 `pointer-events: none`，让拖动/缩放手柄能穿透到窗口。

### 第 3 步：本地控制服务

主进程内置一个最小 HTTP 服务，把 URL 路径当状态名：

```js
// 伪代码
http.createServer((req, res) => {
  const state = decodeURIComponent(req.url.split('?')[0].replace(/^\/+|\/+$/g, ''));
  res.end('ok');
  pushState(state);          // → win.webContents.send('pet-state', state)
}).listen(4747, '127.0.0.1');
```

只听 `127.0.0.1`，不对外。端口随便选一个不冲突的（示例用 4747）。

### 第 4 步：接 Agent hooks

在你的 AI agent（如 Claude Code）的 hook 配置里，把事件映射到 `curl` 调用。
**事件→状态**用 `state-map.md` 的标准接口表。命令形如：

```
curl -s -m 2 http://127.0.0.1:4747/<state> || exit 0
```

- 末尾 `|| exit 0`：挂件没开时 curl 失败也不让 hook 报错刷屏；
- **拉起挂件**：用一条探活兜底——`curl 探活 || <用桌面文件管理器拉起 exe> || exit 0`，
  只有没在跑时才拉。

> ⚠️ **hook 命令用纯命令，不要套 `cmd /c "路径"` 或调 `.bat`**。某些 agent 的 shell 会把
> 这种串改写成"交互式 cmd"：弹横幅、把 stdin 的 JSON 当命令读、根本不执行你的脚本。
> 直接写 `curl ...` / 直接写 exe 路径才靠谱。

### 第 5 步：状态行为分类收进单一信息源

主进程要知道"哪些状态算干活（不睡）"，渲染层要知道"哪些状态是瞬时（自动回 idle）"。
**别把这些分类硬编码散落在两个文件**——收进 theme 配置一处，两边派生：

```json
{
  "default": "idle",
  "busy": ["thinking", "typing", "carrying"],
  "momentary": { "happy": 3500, "error": 3000 },
  "states": { "...": "..." }
}
```

```js
// 主进程：干活态来自 theme，不硬编码
const BUSY = new Set(theme.busy || []);
// 渲染层：瞬时态→停留毫秒，也来自 theme
const MOMENTARY = theme.momentary || {};
```

这样加新状态、改某状态行为，只动 theme 一处，不会漏改其中一边。

### 第 6 步：打包

把主进程、渲染层、theme、assets 打成应用包（如 electron-builder，asar）。
**打包前先杀净所有旧实例进程**，否则输出目录里的文件被占用，打包会失败。

### 第 7 步：部署 + 重启

- **整套覆盖**部署目录，不要只替换 `app.asar`（见下方踩坑②）；
- 用桌面文件管理器（而非 agent 集成终端）拉起 exe，拿干净环境（见踩坑①）；
- 部署后**实际探活 + 驱动几个状态**验证，不要只看打包成功。

## 状态机设计：三类状态，别让兜底逻辑误判

把状态分成三类，运行时逻辑就清楚了：

| 类别 | 例 | 运行时行为 |
|---|---|---|
| **忙态** | thinking / typing / carrying | 正在干活，**永不计自动睡眠**——长构建/测试期间没有新事件 ≠ 空闲 |
| **静止态** | idle / notification | 计空闲超时；超时才睡 |
| **瞬时态** | happy / error | 进入后停留几秒，自动回 default |

两条最容易踩的误判，务必避免：

1. **别用"忙态停滞超时"兜底切 notification**。"typing 进入 N 秒没新事件就当成等用户 →
   切 notification"看似聪明，但长任务（大型构建/测试）几秒内本就没有新 hook，会被反复
   误判成"等待"。**notification 只由真实的"需要用户输入"事件**（权限请求等）驱动。

2. **忙态永不自动睡**。"X 分钟没有新状态就睡觉"对静止态合理，但忙态期间也可能长时间
   没有新 hook（单个长任务），按这个规则会写代码写一半睡着。忙态下直接停掉睡眠计时。

> 经验：这两个 bug 都源于"用时间停滞推断意图"。能用真实事件判断的，就别用超时猜。

## 踩坑清单

### ① `ELECTRON_RUN_AS_NODE`（最大的坑）

很多 AI agent / IDE 自身就是 Electron 应用，会给它派生的终端、子进程注入
`ELECTRON_RUN_AS_NODE=1`。从这种环境直接启动你的 Electron exe，会**继承这个变量** →
exe 被当成普通 Node 跑 → `require('electron')` 返回的是路径字符串而不是模块 →
`app` / `ipcMain` 全是 undefined → 启动即崩。

- **症状**：`node:electron/js2c/node_init` 报错、`ipcMain.on is undefined`；而从桌面双击
  exe 反而正常（干净环境）。
- **解法**：启动前清空这个变量（`set ELECTRON_RUN_AS_NODE=` / `$env:ELECTRON_RUN_AS_NODE=""`），
  或干脆用**桌面文件管理器进程**去拉 exe（它给的是干净的用户会话环境）。

### ② 改了运行时代码，别只替换 `app.asar`

打包后的 `app.asar` 的格式/完整性是**和 exe 绑定**的（取决于 Electron 版本和打包器配置）。
你单独用 asar 工具手打一个新包塞进部署目录 → 应用**启动即崩、零进程、无报错**，极难排查。

- **正解**：改了主进程/渲染层代码后，重新走完整打包，再把**整套**输出（exe + asar 一起）
  覆盖到部署目录。保留你额外放的启动脚本等文件。

### ③ 用文件管理器进程拉 GUI，而不是 `start` / 直接 spawn

从 hook 里拉起挂件时，用桌面文件管理器进程（如资源管理器）去开 exe：

- 它**脱离 hook 的进程树**——hook 结束时不会被连带杀掉（普通 `start` / 直接 spawn 的子
  进程会随 hook 退出被回收）；
- 它给**干净的会话环境**，自动绕开坑①。

### ④ 打包前杀净旧进程

旧实例还跑着时，打包器清不掉输出目录里被占用的文件（locales 等），构建报错。
打包脚本第一步就把所有同名进程杀掉再继续。

### ⑤ 单实例锁

一个桌面挂件应该只开一个。加单实例锁，让"每次 agent 会话都尝试拉起"变成幂等——
已在跑就什么都不做。

## 验收

接好后，按 `state-map.md` §公开目标验证的思路，**验证对象要是你实际看到的运行时**：

1. 探活：控制服务端口有响应；
2. 驱动：手动 `curl .../typing`、`curl .../idle`，看挂件真的在切；
3. 事件联动：在 agent 里实际发消息 / 跑工具，看状态跟着变；
4. 状态分类：长任务期间不误睡、不误切 notification。

源文件只证明你改对了代码；运行时路径才证明部署和映射都对。
