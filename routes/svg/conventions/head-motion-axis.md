# 头部运动轴线约定

> 转头、抬头、低头、眼睛跟随和脸部视线方向，先按同一套 face rig 做轴线，再调表情细节。

## 一句话规则

头部运动不是多张静态帧 crossfade。

正确机制是一只 active SVG 在同一个坐标系里移动和变形：脸部轮廓、浅色脸区域、五官、耳朵、头顶装饰都跟随同一套 facial plane。静态帧淡入淡出只能让端点看起来像，运动中会暴露鼻梁跳、腮红漂、脸边裁切错、斜向视线不成立等问题。

## 先建面部语义锚点

做任何头脸方向前，先标出这些锚点，并让调试页能显示它们：

- 脸部轮廓：左右脸边、下巴、脸顶曲线；
- 黄白分界线：或任何主色区域和浅色脸区域的边界；
- 眼睛中心：视觉中心，不一定等于眼 path 的几何中心；
- 鼻梁：中线和鼻嘴连接关系；
- 嘴：跟随鼻梁，不独立漂；
- 腮红：跟随当前白脸裁切；
- 腮影：贴着当前白脸下缘弧度；
- 耳朵：区分近侧、远侧和被头壳遮挡的关系；
- 头顶：头壳最高点和呆毛 / 角 / 装饰根部。

这些点是语义合同，不是一次性调参截图。后续 `yaw`、`pitch`、斜向混合和 pointer-look 都应该复用它们。

## Pitch 轴线

抬头和低头不是把整张脸上下平移。

抬头时，同一套 facial plane 应该同时发生：

- 黄白分界线上移；
- 白脸向下或向外变多；
- 上半头壳视觉上变少或轻微压低；
- 眼睛、鼻子、嘴、腮红沿 pitch 轴上移；
- 五官可以轻微紧凑，但不要单独改眼睛半径来伪装抬头；
- 耳朵更容易被头壳盖住；
- 头顶装饰可以轻微下压，避免比头壳更抢眼。

低头时方向相反：

- 黄白分界线下移；
- 白脸变小或向内收；
- 上半头壳视觉上变多；
- 五官沿 pitch 轴下移并可轻微舒展；
- 耳朵和头顶装饰更容易露出；
- 下脸可以内收，避免低头时脸颊鼓成另一只角色。

判断标准不是“参数对称”，而是同一只角色的脸部平面在转动。

## Yaw + Pitch Mixer

左右转头和上下抬低头必须能组合。

做 mixer 时至少检查：

- `yaw=left/right` 和 `pitch=up/down` 能合成左上、右上、左下、右下；
- 鼻梁、眼睛、鼻嘴、腮红沿同一 facial plane 走；
- 近侧耳朵不要向外滑，应该围绕稳定中心变化；
- 斜向幅度过猛时，先减组合幅度，不要拆散单轴逻辑；
- 低头斜下露出不该露的主色区域时，优先修白脸几何和裁切，不要靠身体放大或补丁层遮。

一个好的 mixer 应该让斜向视线像算法合成出来的同一只角色，而不是两张端点帧叠在一起。

## 斜向候选帧

斜向方向通过视觉确认后，导出候选锚点包。

每个斜向方向都导出：

- `0%`
- `50%`
- `100%`

`0% / 50% / 100%` 用来防漂移：后续正式动画、循环状态或脚本重写时，必须能对照这三帧检查中途脸边、鼻梁、腮红、腮影和耳朵是否跑偏。

导出帧不应该包含轴线、轮廓 guide、目标影子或其他调试层。调试层留在调试页。

## 裁切和贴脸细节

腮红、腮影、白脸裁切必须跟随同一套脸部几何。

常见稳定结构：

- 白脸 path 是当前帧的真实脸部浅色区域；
- 腮红被当前白脸 path 裁切；
- 腮红可以有自己的平移补偿，但不要跟随脸层纵向压缩到变形；
- 腮影不是独立公式，也不是底边直线采样，它应该像贴在白脸上的曲线；
- 裁切 path 每帧等于同帧白脸 path，不带额外 transform。

如果腮红露出脸边、腮影变直线、白脸和五官分开漂，说明这些细节已经脱离了 face rig。

## 中性帧防跳变

运动中的 `0` 位姿必须和左右 / 上下关键帧来自同一几何模型。

不要让 `yaw=0` 或 `middle` 突然切回 raw master path，而左右关键帧走锚点模型。两套 path 只要鼻梁曲率、点顺序或白脸边界不同，过中线时就会出现“啪一下”的中线跳变。

如果想保留母版静态脸，可以分成两种状态：

- 静态展示：允许使用 raw master path；
- 运动中的中性位姿：继续使用同一套 yaw / pitch / mixer 几何。

这条规则适用于左右摇、低头抓狂、睡觉低头、pointer-look 和任何斜向视线。

## 验收清单

锁定头脸运动前检查：

- 调试页能显示轴线、脸部轮廓 guide 和关键锚点；
- 单轴 `yaw`、单轴 `pitch`、四个斜向方向都看过；
- 每个确认方向都有 `0% / 50% / 100%` 可对照；
- 浏览器循环或指针跟随至少看 30 秒；
- 鼻梁过中线没有跳；
- 眼睛、鼻子、嘴、腮红没有各走各的；
- 腮红始终在白脸裁切内；
- 腮影贴着当前白脸下缘，不漂、不变直；
- 耳朵没有滑出头壳关系；
- 导出文件不包含调试 guide。

同一个中线跳、腮红漂或白脸露馅连续出现两轮，就先停下重审 face rig。继续补局部位移通常只会把两套几何模型缝得更难拆。
